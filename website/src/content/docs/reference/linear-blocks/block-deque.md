`BlockDeque` 是一种基于双端队列（Deque）实现的高性能线性容器。它结合了 `BlockList` 的严格 O(1) 随机访问特性与原生 `Deque` 的双端极致吞吐能力。

通过维持物理块的严格饱满状态，它在保证 O(1) 寻址的同时，实现了头尾两端操作的常数级复杂度。

## 声明

```typescript
export class BlockDeque<T> extends InternalIterable<T>;
```

## 模板参数

| 参数  | 描述             |
| :---- | :--------------- |
| **T** | 存储的元素类型。 |

## 成员函数

### 构造函数

| 方法                             | 描述                                                                                  |
| :------------------------------- | :------------------------------------------------------------------------------------ |
| **constructor(blockSize = 512)** | 构造实例。`blockSize` (即 B) 会被自动向上取整到 2 的幂以优化底层 Deque 的对齐。 |

### 元素访问

| 方法                  | 描述                               | 复杂度 |
| :-------------------- | :--------------------------------- | :----- |
| **at(index)**         | 访问指定位置的元素，支持负数索引。 | **O(1)** |
| **get(index)**        | 访问指定位置的元素。               | **O(1)** |
| **set(index, value)** | 替换指定位置的元素。               | **O(1)** |

`BlockDeque` 的寻址逻辑涉及对头部块偏移量的计算，但本质上仍是纯数学运算，复杂度为严格的 O(1)。

### 容量与状态

| 方法                            | 描述                                                   | 复杂度 |
| :------------------------------ | :----------------------------------------------------- | :----- |
| **length**                      | 返回元素总数。                                         | O(1)   |
| **capacity**                    | 返回当前对象池能容纳的最大元素数（无新分配）。         | O(1)   |
| **isEmpty()**                   | 检查队列是否为空。                                     | O(1)   |
| **resize(newSize, fillValue?)** | 调整容器大小。缩小时截断数据，扩大时填充 `fillValue`。 | O(K)   |
| **reserve(n)**                  | 预分配物理块，确保至少能容纳 $n$ 个元素。              | O(N/B) |
| **shrinkToFit()**               | 释放内存池中闲置的物理块给 GC。                        | O(1)   |
| **clear(reuse = true)**         | 清空容器。如果 `reuse` 为 true，会将物理块回收进对象池。 | O(1) 或 O(N/B) |

### 基础修改器 (Modifiers)

| 方法                     | 描述                               | 复杂度 |
| :----------------------- | :--------------------------------- | :----- |
| **push(...values)**      | 在尾部添加元素。                   | 均摊 **O(1)** |
| **pop()**                | 弹出尾部元素。                     | 均摊 **O(1)** |
| **unshift(...values)**   | 在头部插入一个或多个元素。         | 均摊 **O(1)** |
| **shift()**              | 弹出头部元素。                     | 均摊 **O(1)** |
| **insert(index, value)** | 在指定位置插入，引发智能双向级联位移。 | O(min(idx, N-idx) / B) |
| **delete(index)**        | 删除指定位置元素，引发智能双向级联回收。 | O(min(idx, N-idx) / B) |

`BlockDeque` 的特色在于其**智能级联 (Smart Cascading)**：在中间插入或删除时，它会自动选择向距离最近的端点（头部或尾部）进行位移，从而使平均修改效率优于 `BlockList`。

### 高级与结构操作

| 方法                                | 描述                                                                        | 复杂度 |
| :---------------------------------- | :-------------------------------------------------------------------------- | :----- |
| **splice(start, count?, ...items)** | 切片修改. 支持任意位置的批量删除和插入.                                    | O(K \cdot N/B) |
| **slice(start?, end?)**             | 返回一个指定范围的浅拷贝 `BlockDeque`。                                     | O(K) |
| **concat(...items)**                | 拼接元素、数组、或 `BlockDeque`。<br> 返回一个新的 `BlockDeque`。           | O(K) |
| **reverse()**                       | 原地反转容器内的所有元素。                                                  | O(N) |
| **rebase(newBlockSize?)**           | 重新组织底层块存储。常用于动态修改 `blockSize`。                            | O(N) |
| **toArray()**                       | 将所有元素展开并返回一个原生的 JavaScript 数组。                            | O(N) |

### 迭代器与游标

| 方法                      | 描述                                     |
| :------------------------ | :--------------------------------------- |
| **cursor(index = 0)**     | 返回一个指向指定索引的双向随机访问游标。 |
| **\[Symbol.iterator\]()** | 返回原生迭代器，支持 `for..of` 循环。    |

:::tip[性能提示]
如果你的应用需要频繁地从两端增删元素（双端队列），同时又需要极速的随机索引查询（如渲染超大列表的可见区域），`BlockDeque` 是最理想的选择。
:::
